🔒 Blockchain Security

Understanding why blockchain makes academic certificate forgery mathematically impossible

🔐

SHA-256 Cryptographic Hashing

SHA-256 is a one-way cryptographic function producing a deterministic 256-bit (64 hex char) output. The avalanche effect means even a 1-character change in input produces a completely different hash. It is computationally infeasible to find two inputs that produce the same hash (collision resistance), ensuring certificate fingerprints are unique and unforgeable.

🔗

How Immutability Works

Every block stores the SHA-256 hash of the previous block. If you alter any data in Block N, its hash changes. This makes Block N+1's previousHash field wrong. Which changes Block N+1's hash. Which breaks Block N+2. And so on. One modification invalidates the entire subsequent chain — detectable in milliseconds.

🚫

What Happens When Data Is Modified

If an attacker changes even a single character of a certificate (e.g., grade from C to A): (1) The certificate's SHA-256 hash no longer matches the stored certHash. (2) The block's hash changes, breaking the chain linkage. (3) All four verification stages fail with an exact error message identifying the tampered block.

⛏️

Why Proof-of-Work Is Used

Without PoW, an attacker could instantly re-mine all blocks after modifying a certificate. PoW requires finding a nonce such that the block's hash starts with N leading zeros — requiring ~16N hash computations on average. This makes retroactive chain rewriting computationally expensive and time-consuming, serving as a deterrent against forgery.

🌐

Ethereum Decentralization

CertChain now runs on the real Ethereum Sepolia blockchain — secured by thousands of globally distributed Proof-of-Stake validators. An attacker would need to control >33% of all staked ETH to manipulate the chain. Every certificate is backed by an immutable on-chain transaction verifiable at sepolia.etherscan.io.

🛡️

Non-Repudiation

Once a certificate is mined into the blockchain, the issuing institution cannot deny its issuance — the block with the exact timestamp, certificate data, and block hash is permanently recorded. Conversely, a certificate that does not appear on the chain cannot be claimed as genuine — providing mathematical proof of both existence and non-existence.
